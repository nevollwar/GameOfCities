# Игровая система «Игра в города» (Курсовой проект)

Данный программный комплекс представляет собой единое решение, реализованное в рамках курсовой работы по дисциплине **«Основы программирования»**. Программа моделирует интеллектуальную лингвистическую игру между человеком и компьютерным оппонентом, используя современные архитектурные паттерны и алгоритмы обработки текстовой информации.

## Функциональные возможности

Комплекс разделен на три ключевых модуля анализа и управления:

### 1. Лингвистический анализ и валидация (Проект Logic)
*   **Верификация данных**: Проверка существования введенного города в эталонной базе (более 50 000 записей).
*   **Алгоритм обработки окончаний**: Автоматическое определение целевой буквы с учетом исключений русского языка (игнорирование символов `Ь`, `Ы`, `Ъ`, `Й`).
*   **Контроль уникальности**: Использование хеш-таблиц (`HashSet`) для предотвращения повторного использования городов в рамках одной сессии (сложность поиска $O(1)$).

### 2. Игровой искусственный интеллект
*   **Вариативность ответов**: Реализация алгоритма случайного выбора (`Randomized Selection`) из пула доступных городов для исключения линейности игрового процесса.
*   **Многоуровневая сложность**: Динамическое управление игровым состоянием через три пресета:
    *   **Легкий**: Лимит времени 60 секунд.
    *   **Средний**: Лимит времени 30 секунд.
    *   **Сложный**: Лимит времени 15 секунд.

### 3. Инфраструктура интерфейса (Проект UI)
*   **Адаптивная верстка**: Программная реализация графического интерфейса с использованием якорей (`Anchors`) и докинга (`Docking`), обеспечивающая корректное отображение в полноэкранном режиме.
*   **Событийная модель**: Обработка игровых триггеров (таймер, ввод с клавиатуры, системные уведомления) в реальном времени.

*   **Интерфейс**: Windows Forms (Dynamic UI)
*   **Тестирование**: MSTest Framework
*   **Инструментарий**: Visual Studio 2022
*   **Система контроля версий**: Git (GitHub)

## Формат данных
Программа считывает лексическую базу из текстового файла `cities.txt`.  
**Структура данных**: Одно название города на каждой строке.  
**Препроцессинг**: При загрузке выполняется автоматическая очистка от лишних пробелов (`Trim`) и нормализация регистра.

## Исследование алгоритмической сложности
В рамках проекта проведено сравнение эффективности структур данных:
*   Использование `HashSet<string>` для хранения истории ходов позволило сократить время проверки на повтор с $O(N)$ до $O(1)$ в среднем случае, что критично при работе с большими объемами данных.
*   Алгоритм поиска ответа бота оптимизирован через LINQ-запросы к типизированным спискам.

## Тестирование (Unit Testing)
Система покрыта модульными тестами в проекте `GameOfCities.Tests`. Верифицированы:
*   Математическая точность алгоритма выбора следующей буквы.
*   Устойчивость системы к некорректному вводу (пустые строки, спецсимволы).
*   Логика обработки исключений при отсутствии файла базы данных.

## Инструкция по запуску
1.  Клонируйте репозиторий: `git clone https://github.com/ваш_логин/GameOfCities.git`
2.  Откройте файл решения `GameOfCities.sln` в Visual Studio 2022.
3.  Выполните сборку решения (**Build Solution**).
4.  Убедитесь, что файл `cities.txt` находится в директории `bin/Debug/net8.0/` проекта UI.
5.  Запустите проект `GameOfCities.UI`.
